Datapaws Monitoring Extension

Compatible with Chrome, Edge & Chromium browsers

Datapaws Monitoring Extension is a browser extension that collects console messages, page load times, and render timings, sending them to Datadog for monitoring and analysis. Compatible with Chrome, Microsoft Edge, and other Chromium-based browsers.
Every metric is sent as custom-metric.

GitHub project page: https://github.com/gcaglio/EdgeExt-DataPaws
✨ New in this version: Now uses modern PerformanceObserver API for Core Web Vitals. Memory metrics have been removed as they used deprecated Chrome-only APIs.

The extension collects and sends the following metrics:
  • datapaws.chrome.pageLoadTime: Total page load time.
  • datapaws.chrome.domContentLoadTime: Time taken to load the DOM.
  • datapaws.chrome.renderTime: Time taken to render the page.
  • datapaws.chrome.totalBlockingTime: Total blocking time.
  • datapaws.chrome.ttfb: Time to First Byte.
  • datapaws.chrome.firstContentfulPaint: First Contentful Paint (FCP) - Core Web Vital.
  • datapaws.chrome.largestContentfulPaint: Largest Contentful Paint (LCP) - Core Web Vital.
  • datapaws.chrome.cumulativeLayoutShift: Cumulative Layout Shift (CLS) - Core Web Vital.
  • datapaws.chrome.consoleError: Console, window and "unhandled promise rejections" errors.
⚠️ Deprecated metrics removed: usedJSHeapSize and totalJSHeapSize have been removed as they used the deprecated performance.memory API which is Chrome-only and non-standard.

All metrics include additional tags such as:
  • site_fqdn: Hostname of the site.
  • resolution: Screen resolution.
  • browser: Browser user-agent.
  • hostname: Device name.
  • browser_deviceid: a random, one time generated deviceId that persist in the local browser storage, to allow a "per browser instance" grouping, to allow event correlation and problem determination.
  • navigation_type: Type of navigation (navigate, reload, back_forward).
  • connection_type: Connection type (4g, 3g, wifi, etc).

To allow the extension to send data to your Datadog account, you need to configure:
  • Datadog API URL. Eg: https://api.datadoghq.com, https://api.datadoghq.eu, etc
  • Datadog API key. You can get it logging in into your account, hovering onto your profile icon (lower left) and select Organization>Api Keys
  • Hostname. You need to define one 'hostname' to correlate all the metrics.
  • Custom tags (optional). You can add custom tags to all metrics for better filtering and grouping.
Example:
department=IT;environment=production;cost_center=123456;region=EU;team=WebDev;
